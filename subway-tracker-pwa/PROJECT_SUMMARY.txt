🛠️ TECHNICAL ARCHITECTURE

• MTA GTFS-realtime - train arrivals (requires API key)
• Camera API - car number scanning (optional)
• Notifications API - ride confirmations

BACKEND (OPTIONAL):
• PostgreSQL database schema provided
• REST API endpoints documented
• User authentication ready
• Cloud sync support


🚀 QUICK START GUIDE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. GET MTA API KEY
   → Visit: https://api.mta.info/
   → Register for free developer account
   → Copy your API key

2. CONFIGURE API KEY
   → Open: src/js/mtaService.js
   → Line 3: this.apiKey = "YOUR_MTA_API_KEY"
   → Replace with actual key

3. SERVE LOCALLY
   # Python option:
   cd subway-tracker-pwa
   python -m http.server 8000

   # Node.js option:
   npx http-server -p 8000

   → Open: http://localhost:8000/public/

4. TEST FEATURES
   ✓ Enable location permissions
   ✓ View nearby stations
   ✓ Log a test ride
   ✓ Check stats dashboard

5. INSTALL AS PWA
   • Mobile: Tap "Add to Home Screen"
   • Desktop: Click install icon in address bar

🔧 CUSTOMIZATION TIPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Add More Stations:
→ Edit: src/js/locationService.js
→ Array: this.stations = [...]
→ Add: { id, name, lat, lng, lines }

Change Theme Colors:
→ Edit: src/css/styles.css
→ Variables: :root { --primary-color: ... }

Adjust Ride Duration:
→ Edit: src/js/statsService.js
→ Variable: this.avgRideDuration = 25  // minutes

Car Model Logic:
→ Edit: src/js/dbService.js
→ Method: inferCarModel(carNumber)

Enable OCR Scanning:
→ Add to index.html:
  <script src="tesseract.min.js"></script>
→ Implement in: app.js → captureCarNumber()

💡 PRO TIPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Location Permissions
   → Request on first launch with clear explanation
   → Fallback to manual station selection if denied

2. Battery Optimization
   → Don't continuously watch location
   → Only update when app is active

3. Offline UX
   → Show "offline" badge when no connection
   → Queue rides for sync with visual indicator
   → Never block user from logging rides

4. Data Validation
   → Check car numbers match 4-digit pattern
   → Verify station IDs exist before logging
   → Limit note length to 50 characters

5. User Feedback
   → Always confirm ride logged (notification)
   → Show loading states for API calls
   → Clear error messages if something fails


🐛 COMMON ISSUES & FIXES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue: "Location not detected"
Fix: Enable browser location permissions
Fix: Use HTTPS (required for geolocation)

Issue: "Service Worker not caching"
Fix: Clear browser cache and reload
Fix: Check sw.js is in /public/ root

Issue: "App won't install"
Fix: Must serve over HTTPS (or localhost)
Fix: Ensure manifest.json is valid JSON

Issue: "Rides not syncing to backend"
Fix: Check backend API URL is correct
Fix: Verify CORS headers on API
Fix: Check network tab for 401/403 errors

Issue: "IndexedDB quota exceeded"
Fix: Implement data retention policy
Fix: Delete old rides after 1 year
Fix: Compress ride data format
